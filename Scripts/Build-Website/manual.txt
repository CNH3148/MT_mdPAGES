# Build-Website 腳本使用手冊

## 腳本：build_website.py

### 核心功能
將本地端基於 Flask/Node.js 伺服器的 `app`，轉換成純靜態網站（適合部署至 GitHub Pages）。
本腳本會：
1. 複製所有靜態資源與圖片。
2. 呼叫 `cache_builder` 將 Markdown 題庫編譯成 JSON。
3. 自動修改 `app_v2.js`，將對後端 API 的呼叫註解掉並導向本地快取。
4. 修改 HTML 中的相對路徑，避免在 GitHub Pages 子目錄中發生 404 錯誤。

### 執行方式
```bash
python Scripts/Build-Website/build_website.py [參數]
```

### 所需參數 (Arguments)
- `--app-dir`: 原始應用程式的目錄位置（預設為 `app`）。
- `--data-dir`: Markdown 題庫的目錄位置（預設為 `app/data_MD`，未來可設定為 `data_MD`）。
- `--output-dir`: 產生的靜態網站存放目錄（預設為 `data_MD_website`）。

### 輸入/輸出範例
**輸入環境**：
- `app/` (包含 public/, data_MD/, md_engine/)

**執行指令**：
```bash
python Scripts/Build-Website/build_website.py
```

**輸出結果**：
- 成功產生 `data_MD_website/` 目錄。
- 內部包含轉換後的 JSON 檔案 (`data_cache/`)、靜態網頁與調整過路徑的 JavaScript/HTML 檔案。
