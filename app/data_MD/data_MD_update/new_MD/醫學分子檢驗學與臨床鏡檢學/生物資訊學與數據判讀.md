## 核心趨勢與高頻考點
本單元的核心在於測驗「從原始數據到臨床意義」的完整分析流程。考題趨勢顯示，單純記憶資料庫名稱已非重點，而是更強調**分析流程的邏輯性**、**工具選擇的目的性**以及**國際命名標準的嚴謹性**。
1.  **標準化 (Normalization) 的必要性**：無論是微陣列 (Microarray) 還是次世代定序 (NGS) 的表現量分析 (RNA-seq)，比較樣本前都必須進行「標準化」。這是高頻考點，因為它是確保數據可比性的基礎。其邏輯鏈為：**實驗操作產生系統性誤差 (如上樣量不同) → 原始數據訊號/讀取數不具直接可比性 → 必須進行【數據標準化】→ 才能準確比較樣本間的基因相對表現量**。
2.  **HGVS 基因變異命名標準 (新興高頻)**：近年強勢考點，測驗考生是否具備撰寫及解讀標準臨床報告的能力。必須熟記前綴詞（`g.`、`c.`、`p.`、`r.`），並深刻理解各種變異（如 insertion）在紀錄位置上的邏輯限制。
3.  **變異判讀的黃金流程**：考題圍繞著如何從一堆 ATCG 序列中，找出有意義的突變。其邏輯鏈為：**患者檢體定序 (Raw Reads) → 與【參考基因組 (Reference Genome)】進行序列比對 (Alignment) → 找出差異點 (Variant Calling) → 透過資料庫進行【基因註解 (Annotation)】→ 判斷變異的臨床意義 (如：致病性 Pathogenic vs. 良性 Benign)**。
4.  **精準選擇資料庫與 BLAST 演算法**：
    *   **資料庫性質**：想找序列本身用 **GenBank/RefSeq/DDBJ/EMBL**（核酸）或 **UniProt**（蛋白質）；想查序列與遺傳疾病的關聯，則必須用 **OMIM**。
    *   **BLAST 工具選擇**：需根據手上的 Query 與目標 Database 的「分子性質」選擇。例如，純 DNA 對純 DNA 用 `blastn`。
## 未來考點預測與易混淆陷阱
*   **HGVS 命名陷阱 1（不存在的前綴）**：題目會捏造前綴詞來混淆，例如 `i.` (宣稱代表 intron/exon)，這是**絕對錯誤**的。內含子突變通常依附在 `c.` (coding DNA) 加上偏移量來表示，或直接使用 `g.` (genomic DNA)。
*   **HGVS 命名陷阱 2（插入突變 insertion 的位點標示）**：插入突變是發生在兩個核苷酸「之間」，因此位點數字**必須是相鄰的兩個數字**（例如 `c.1253_1254insC`）。若看到跨越範圍大於 2 的標示（例如 `c.324_329insGAAGT`），在邏輯上絕對是錯誤的，那應該稱為缺失插入 (delins) 而非單純插入。
*   **BLAST 演算法的混淆**：
    *   `blastn` = Nucleotide 對 Nucleotide（最直觀的基因比對）。
    *   `blastx` = 拿 Nucleotide 翻譯成 Protein 後，去對 Protein 資料庫（找未知基因的功能）。
    *   `tblastx` = 雙方都從 Nucleotide 翻譯成 Protein 後比對（找遠緣同源基因）。
*   **「分析敏感性」 vs 「診斷敏感性」在篩檢的應用**：分子診斷若作為「篩選試驗 (Screening test)」，首要目標是不漏掉任何陽性個案，因此檢驗本身的偵測極限要夠低，必須追求**高「分析敏感性」 (Analytic sensitivity)**。
