# 專案專屬架構與規範
本專案為自動化腳本與工具庫，嚴格劃分兩個核心工作區：`Scripts/`（腳本庫）與 `Temps/`（中繼資料區）。你在此專案的所有行為必須絕對遵守以下規範。

# 1. 腳本庫 (Scripts) 管理規範
所有你撰寫、修改或執行的自動化腳本、工具程式，都必須存放在專案根目錄的 `Scripts/` 資料夾下：
- **命名原則 (Verb-Noun)**：所有分類資料夾與腳本名稱必須採用英文，並嚴格遵循「動詞-名詞」的設計精神（例如：`Fetch-Data`, `Clean-Format`）。以保持高可讀性為最高原則，限制使用 2 到 3 個單字組合而成，單字之間以連字號 (`-`) 或底線 (`_`) 區隔，避免過度冗長。
- **全域索引維護 (`Scripts/scripts_index.txt`)**：這份文件是專案的「大腦」，你必須在每次新增或修改分類資料夾/腳本時更新它。內容需包含：(1) 每個分類資料夾的用途概述。(2) 每個腳本的核心功能與歷史使用紀錄。
- **區域手冊維護 (`Scripts/<分類>/manual.txt`)**：每個分類資料夾內必須有一份詳細說明書，只要該目錄下有腳本變動，就必須同步更新。內容需包含當前資料夾內所有腳本的詳細用途、執行方式、所需參數（Arguments）及輸入/輸出格式範例。

# 2. 中繼資料區 (Temps) 管理規範
所有任務執行過程中產生的暫存檔、中繼資料、下載的暫時性檔案，嚴禁污染工作區，必須統一存放在 `Temps/` 資料夾下。
- **結構對齊**：`Temps/` 的內部子資料夾應與 `Scripts/` 的 Verb-Noun 分類邏輯對齊（例如產生自 `Scripts/Fetch-Data/` 的暫存檔應存放在 `Temps/Fetch-Data/`），確保中繼檔案能快速與產生它的腳本關聯，以利後續清理與追溯。

# 3. 本專案開發 SOP 附加條款
- **先查閱索引**：開發任何新腳本前，必須先查閱 `Scripts/scripts_index.txt`。
- **同步更新文件**：腳本開發完成且確認無誤後，**必須立刻**更新對應的 `manual.txt` 與全域 `scripts_index.txt`。
- **指定套件**：撰寫 Python 腳本時，必須使用 `argparse` 解析參數。
